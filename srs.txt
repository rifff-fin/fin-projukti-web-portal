Software Requirements Specification (SRS)Project: Fin Projukti Institutional WebsiteVersion: 1.0Date: March 26, 2026Status: Initial Draft1. Introduction1.1 PurposeThe purpose of this document is to define the functional and non-functional requirements for the Fin Projukti official website. This site serves as the primary digital touchpoint for a BRAC University student-led startup providing high-end Web and AI solutions.1.2 ScopeThe website will showcase Fin Projukti’s technical arsenal (MERN, AI, C++, Java, etc.), list professional services, display a project portfolio, and facilitate client acquisition through an automated contact portal.1.3 Definitions & AcronymsMERN: MongoDB, Express, React, Node.js.CTA: Call to Action (e.g., "Automate My Business").UI/UX: User Interface / User Experience.Next-Gen: High-performance, future-ready technology.2. Overall Description2.1 Product PerspectiveThis is a standalone web application designed to move Fin Projukti from a service-based agency to an institutional tech giant. It must compete visually with premium global digital agencies.2.2 Product FunctionsBrand Storytelling: Communicating the "Fin" philosophy (ending manual limits).Lead Generation: Automated intake of client requirements via a custom form.Technical Authority: Displaying a "Tech DNA" marquee to prove engineering depth.Social Proof: Highlighting the BRACU foundation and student-led innovation.2.3 User Classes and CharacteristicsPotential Clients: Business owners looking to move from social media DMs to professional web platforms.Talent/Recruits: Developers looking to join a high-growth startup.Partners/Investors: Entities interested in the "Big Tech" vision of the founders.2.4 Design ConstraintsColor Palette: Strict adherence to #000000 (Black), #CCCCCC (Grey), and #FF6600 (Orange).Performance: Must achieve a Lighthouse score of $90+$ for speed and SEO.3. System Features3.1 Global Navigation (Sticky Navbar)Requirement: Glassmorphism effect with the Fin Projukti orange circle logo.Links: Home, Services, Work, About (DNA), and a primary "Start Project" button.3.2 Hero Section (The Hook)Requirement: Dynamic typography featuring the slogan: "The End of Manual Limits. The Beginning of AI Growth."Feature: Subtle background animations (Framer Motion) and high-contrast orange CTAs.3.3 Service Pillars (The Offering)Requirement: Interactive grid or vertical list showing 01. MERN Ecosystems, 02. AI & Automation, 03. Industrial Engineering.Hover Effect: Border transitions from Grey to Orange.3.4 Technical DNA MarqueeRequirement: An infinite-scrolling marquee showcasing technologies like Python, C++, Java, React, and AI Stacks to build technical trust.3.5 Lead Intake System (Contact Portal)Requirement: A minimalist form with validation.Integration: Successful submission triggers an automated response and a redirect to the official WhatsApp Business API.4. External Interface Requirements4.1 User InterfacesResponsive Design: Optimized for Mobile (70% of traffic) and Ultra-wide Desktop monitors.Dark Mode: Default and permanent state (no light mode toggle to maintain "Big Tech" aesthetic).4.2 Software InterfacesFrontend: React.js / Next.js with Tailwind CSS.Animation Library: Framer Motion for premium scroll effects.Deployment: Vercel or Netlify for high-speed edge delivery.5. Non-Functional Requirements5.1 PerformanceRequirement: Initial page load under $1.5$ seconds on 4G connections.5.2 SecurityRequirement: SSL encryption (HTTPS) and protection against basic form-spamming using honeypots or reCAPTCHA.5.3 ScalabilityRequirement: Modular component architecture allowing for new case studies and services to be added without breaking the layout.6.
